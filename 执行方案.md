## 四、链码设计（chaincode/copyright/）

链码三个源文件已创建完毕：

| 文件 | 说明 |
|---|---|
| `chaincode/copyright/main.go` | 入口，注册 CopyrightContract 并启动链码 |
| `chaincode/copyright/model.go` | 数据结构：MusicWork、License、DisputeRecord、VerifyResult |
| `chaincode/copyright/contract.go` | 全部业务逻辑实现 |

**model.go 数据结构说明**：
- `MusicWork`：作品，Status 枚举 `ACTIVE / TRANSFERRED / DISPUTED`
- `License`：授权，Status 枚举 `ACTIVE / REVOKED`，MaxUsage=0 表示不限次
- `DisputeRecord`：争议存证，Status 枚举 `PENDING / RESOLVED`
- `VerifyResult`：授权核验返回体，含 valid、reason、license 字段

### 4.2 链码函数（contract.go）

| 函数名 | 类型 | 参数 | 说明 |
|---|---|---|---|
| `RegisterWork` | Invoke | workID, title, artist, ownerID, fileHash, description, genre | 版权存证，检查 workID 唯一性，时间戳由 stub 写入 |
| `QueryWork` | Evaluate | workID | 查询单个作品 |
| `QueryWorksByOwner` | Evaluate | ownerID | CouchDB 富查询，按所有人查全部作品 |
| `SearchWorks` | Evaluate | keyword | CouchDB 富查询，按 title/artist 模糊匹配 |
| `GrantLicense` | Invoke | licenseID, workID, grantorID, licenseeID, licenseType, startDate, endDate, maxUsage | 发放授权，校验 grantorID == work.ownerID |
| `VerifyLicense` | Evaluate | workID, licenseeID | **核验某用户对某作品是否持有当前有效授权**，返回授权详情或"无授权" |
| `RevokeLicense` | Invoke | licenseID, revokerID | 撤销授权，校验 revokerID == grantor |
| `QueryLicensesByWork` | Evaluate | workID | 查询某作品所有授权 |
| `QueryLicensesByLicensee` | Evaluate | licenseeID | 查询某用户持有的所有授权 |
| `RecordUsage` | Invoke | licenseID | 记录一次使用，usedCount++，若超 maxUsage 则拒绝 |
| `TransferCopyright` | Invoke | workID, fromID, toID | 版权转让，校验 fromID == work.ownerID |
| `FileDispute` | Invoke | disputeID, workID, claimantID, evidence | 提交版权争议存证 |
| `GetHistory` | Evaluate | workID | 获取作品全部历史变更 |
| `VerifyFileHash` | Evaluate | workID, fileHash | 验证上传文件哈希与链上记录是否一致，返回 true/false |

### 4.3 关键实现细节

**VerifyLicense 逻辑**（最重要，实用度核心）：
```
1. 查找该 workID 下 licenseeID 匹配的所有 License
2. 逐条判断：Status == ACTIVE
3. 判断当前时间是否在 [StartDate, EndDate] 范围内
4. 若 maxUsage > 0，判断 usedCount < maxUsage
5. 全部满足则返回授权详情；否则返回 {valid: false, reason: "..."}
```

**SearchWorks 富查询**（CouchDB selector）：
```go
query := fmt.Sprintf(`{
    "selector": {
        "$or": [
            {"title": {"$regex": "(?i)%s"}},
            {"artist": {"$regex": "(?i)%s"}}
        ]
    },
    "limit": %d,
    "skip": %d
}`, keyword, keyword, limit, skip)
return stub.GetQueryResult(query)
```

**复合键索引**：同时建立 `owner~work`、`licensee~license`、`work~license` 三种复合键，支持 O(1) 范围查询，避免全表扫描。

**时间戳**：所有写操作的时间字段统一用 `stub.GetTxTimestamp()` 写入，禁止使用客户端传入时间。

## 六、核心服务实现

### 6.1 Fabric Gateway 封装（service/fabric.go）

```go
type FabricClient struct {
    contract *gateway.Contract
}

func NewFabricClient(cfg *config.FabricConfig) (*FabricClient, error) {
    clientConnection := newGrpcConnection(cfg)
    id := newIdentity(cfg)
    sign := newSign(cfg)

    gw, err := gateway.Connect(
        gateway.WithClientConnection(clientConnection),
        gateway.WithIdentity(id),
        gateway.WithSign(sign),
    )
    if err != nil {
        return nil, err
    }

    network := gw.GetNetwork(cfg.Channel)
    contract := network.GetContract(cfg.Chaincode)
    return &FabricClient{contract: contract}, nil
}

func (c *FabricClient) Submit(fn string, args ...string) ([]byte, error) {
    return c.contract.SubmitTransaction(fn, args...)
}

func (c *FabricClient) Evaluate(fn string, args ...string) ([]byte, error) {
    return c.contract.EvaluateTransaction(fn, args...)
}
```

### 6.2 文件哈希计算（utils/hash.go）

```go
func SHA256File(file multipart.File) (string, error) {
    h := sha256.New()
    if _, err := io.Copy(h, file); err != nil {
        return "", err
    }
    return hex.EncodeToString(h.Sum(nil)), nil
}
```

音频文件**不存服务器**，只存 SHA-256 哈希上链。用户日后可上传同一文件进行哈希比对来证明文件未被篡改。

### 6.3 存证证书生成（utils/certificate.go）

使用 `github.com/jung-kurt/gofpdf` 生成 PDF，内容包含：
- 作品名称、创作者、版权人
- 存证时间（链上 tx timestamp）
- 文件 SHA-256 哈希值
- Fabric 交易 ID（可在区块链浏览器核查）
- 二维码（链接到本系统查询页面）

这是创新性的核心加分点：生成具有法律参考价值的版权存证证书。

### 6.4 JWT 中间件（api/middleware/auth.go）

```go
func AuthMiddleware(secret string) gin.HandlerFunc {
    return func(c *gin.Context) {
        tokenStr := c.GetHeader("Authorization")
        if !strings.HasPrefix(tokenStr, "Bearer ") {
            c.JSON(http.StatusOK, utils.Error(1002, "未登录"))
            c.Abort()
            return
        }
        token, err := jwt.Parse(strings.TrimPrefix(tokenStr, "Bearer "), ...)
        if err != nil || !token.Valid {
            c.JSON(http.StatusOK, utils.Error(1002, "Token 无效"))
            c.Abort()
            return
        }
        claims := token.Claims.(jwt.MapClaims)
        c.Set("userID", claims["userID"])
        c.Next()
    }
}
```

### 6.5 分页处理（通用）

所有列表接口支持 `?page=1&size=10`，后端解析后传给链码的 limit/skip 参数。

## 十一、实现顺序

1. **搭 Fabric 网络**：跑通 bootstrap.sh，确认 CouchDB 和 peer 容器正常，通道建好。
2. **写链码**：model.go → contract.go，重点实现 `VerifyLicense` 和 `SearchWorks`，用 `shimtest.MockStub` 单元测试。
3. **部署链码**：deploy.sh 跑通，用 `peer chaincode invoke` 手工验证各函数。
4. **后端骨架**：main.go + router.go + 统一响应 + 错误码 + JWT 中间件 + CORS。
5. **Fabric 服务层**：service/fabric.go 封装 Gateway，先跑通一个 `RegisterWork` 调用。
6. **用户模块**：注册/登录，MySQL 联调。
7. **版权模块**：存证 → 查询 → 搜索 → 哈希验真 → 历史，逐接口联调。
8. **授权模块**：发放 → 核验 → 记录使用 → 撤销，重点保证 `VerifyLicense` 逻辑正确。
9. **创新功能**：存证证书 PDF 生成 + 版权争议存证接口。
10. **联调自测**：Postman 走完完整业务流：注册→存证→发授权→核验→记录使用→生成证书。

## 十三、注意事项

1. **CouchDB 必须启用**：`SearchWorks` 和 `QueryWorksByOwner` 使用富查询，LevelDB 不支持，必须在 docker-compose 配置 CouchDB。
2. **链码时间戳**：所有时间字段用 `stub.GetTxTimestamp()` 写入，不接受客户端传入，防伪造。
3. **哈希存证不存文件**：降低链存储压力，文件验真通过重新计算哈希比对实现。
4. **身份模型说明**：演示系统中链码通过 `ownerID` 参数做业务权限校验，真正的身份认证在后端 JWT 层完成。生产环境应使用 Fabric CA。
5. **授权核验是系统核心价值**：`GET /api/v1/license/verify` 模拟了流媒体平台在播放音乐前核验版权授权的真实场景，这是整个系统实用度最高的功能点。
